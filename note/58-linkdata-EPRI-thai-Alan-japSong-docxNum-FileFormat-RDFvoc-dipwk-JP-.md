1. ขาดมาตรฐานรูปแบบวิธีการบันทึก (IRI, RDF)

ข้อมูลที่ควรต้องจัดเก็บรวบรวมเพื่อการติดตามผลมีหลายประเภท
ข้อมูลบางอย่างสามารถกำหนดรูปแบบชัดเจนได้ง่ายเป็นรูปธรรม 
เช่น วันที่ จำนวนเงิน ระยะเวลา ชื่อคน ชื่อโครงการ เป็นต้น
ข้อมูลบางอย่างไม่สามารถกำหนดรูปแบบได้เนื่องจากเป็นนามธรรม
จำเป็นต้องตีความ จำเป็นต้องอธิบาย เช่น ประเภทโครงการ สถานะโครงการ
ความก้าวหน้าโครงการ ความเสี่ยงโครงการ เป็นต้น
การบันทึกข้อมูลต้องอาศัยการคิดวิเคราะห์ และคำนึงถึงการนำไปใช้ในการประเมินผล
เช่น สถานะโครงการ อาจบันทึกเป็นคำอธิบายสถานะของโครงการ 
ถ้าข้อมูลที่บันทึกสถานะโครงการดังกล่าวกำหนดให้เป็น
'ก่อนเริ่มโครงการ' 'เริ่มโครงการ' 'สิ้นสุดโครงการ'
ก็จะทำให้สามารถประเมินผลได้ว่า โครงการอยู่ในสถานะไหน
แต่เมื่อกำหนดเพียง 3 สถานะข้างต้น ก็อาจมีข้อจำกัดในประเมินผลโครงการในบางเรื่อง
เช่น จำเป็นต้องบันทึกวันที่เริ่มต้นวางแผนโครงการ วันที่เริ่มโครงการ วันที่สิ้นสุดโครงการ

แต่เมื่อเวลาผ่านไปก็อาจพบว่าการแบ่งสถานะเป็น 3 สถานะนั้นไม่ละเอียดพอ
ไม่สามารถประเมินผลมิติอื่นๆ ที่สามารถจำเป็นต้องพิจารณาเพิ่มเติม จึงเกิดความต้องการ
ในการบันทึกข้อมูลเพิ่มเติม

เนื่องจากในการคิดวิเคราะห์เพื่อกำหนดชุดข้อมูลเพื่อการบันทึกรวบรวมเพื่อใช้ในการติดตาม
นั้น ได้รับอิทธิพลจากหลายปัจจัย เจ้าหน้าที่ผู้คิด ผู้บริหารที่คิด บริบทของการคิดในแต่ละสถานการณ์
ที่แตกต่างกัน ดังนั้น เป็นธรรมดาอยู่เองที่ ชุดข้อมูลที่เหมาะสมในแต่ละช่วงเวลา
แต่ละบริบทของการคิดนั้นมีความแตกต่างกัน แต่ละชุดข้อมูลที่คิดขึ้นต่างมีข้อดี และข้อด้อยแตกต่างกัน
ไป ดังนั้นจึงมีความจำเป็นต้องหา มาตรฐานรูปแบบวิธีการที่มีความยืดหยุ่น แต่ยังรักษา
ทั้งความชัดเจนของข้อมูลได้อย่างมีประสิทธิภาพ

ในการนี้ผู้รับการประเมินได้ศึกษา มาตรฐานรูปแบบวิธีการที่เหมาะสมในการบันทึกข้อมูลติดตามผล
เรียกว่า Resource Description Framework หรือ อาร์ดีเอ็ฟ (RDF) ซึ่งเป็นมาตรฐานรูปแบบวิธีการ
ที่ประกาศโดยองค์กรสากลได้อินเทอร์เน็ตคือ World Wide Web Consortium หรือ ดับเบิลยูสามซี (W3C)
(https://www.w3c.org) ซึ่งได้รับการประกาศใช้อย่างเป็นทางการ ในปี ค.ศ.1999
คิดค้นโดย เซอร์ ทิม เบิร์นเนอร์ ลี (Sir Tim Berner Lee) ซึ่งเป็นคิดค้น World Wide Web 
ที่ใช้กันอยู่ในปัจจุบัน มาตรฐาน ​RDF ได้รับการปรับปรุงล่าสุดเมื่อ ปี ค.ศ.2014

มาตรฐาน RDF ถูกใช้ในการจัดเก็บข้อมูลในรูปแบบที่ใกล้เคียงกับภาษามนุษย์ มีโครงสร้างพื้นฐาน
ที่ประกอบไปด้วยประโยค (Statement) ซึ่งแต่ละประโยคประกอบด้วย 3 ส่วนได้แก่ ประธาน (Subject)
กริยา (Predicate or Verb or Property) และ กรรม (Object)
มาตรฐาน RDF เป็นรูปแบบการจัดเก็บข้อมูลที่เหมาะสมกับการจัดเก็บข้อมูลทุกประเภท
มีความยืดหยุ่น สามารถปรับปรุงรูปแบบให้เหมาะสมกับ รูปแบบวิธีการคิดที่หลากหลาย
บริบทเกี่ยวข้องที่หลากหลาย อีกทั้งยังสามารถบูรณาการเชื่อมโยงความหมายข้อมูลได้อย่างเป็นระบบ
ไม่มีความจำเป็นต้องกำหนดรูปแบบที่คงตัว

ที่ผ่านมา RDF ถูกใช้ในการจัดเก็บข้อมูลอย่างกว้างขวางในหลายวงการ
RDF ใช้จัดเก็บข้อมูลทางการแพทย์อย่างกว้างขวาง ข้อมูลโรค ข้อมูลเชื้อโรค ข้อมูลวิธิีการรักษา
ทำให้ข้อมูลที่ถูกจัดเก็บในรูปแบบดังกล่าว สามารถนำไปประยุกต์ใช้ได้อย่างง่ายดายและกว้างขวาง
เกิดประโยชน์กับสถานพยาบาล ผู้ป่วย และคนจำนวนมาก
RDF ใช้ในการจัดเก็บข้อมูลผลการทดลอง ผลงานวิจัยทางวิทยาศาสตร์ อย่างกว้างขวาง ทำให้
การแลกเปลี่ยนและใช้ข้อมูลร่วมกันสามารถทำได้อย่างกว้างขวาง

การประเมินผล

การจัดเก็บข้อมูลติดตามผลมีความจำเป็นต้องถูกนำไปใช้ในการประเมินผล
การบันทึกลงในกระดาษ ทำให้การประเมินผลไม่มีประสิทธิภาพ
ทั้งในกระบวนการบันทึก กระบวนการรวบรวมประมวล และกระบวนการนำไปคิดวิเคราะห์
ดังนั้นการจัดเก็บข้อมูลจะต้องใช้เครื่องมือที่เป็นซอฟต์แวร์คอมพิวเตอร์

การจัดเก็บข้อมูลโดยใช้คอมพิวเตอร์ จำเป็นต้องใช้ซอฟต์แวร์เป็นเครื่องมือในการบันทึกข้อมูล
และจำเป็นต้องใช้ระบบจัดเก็บข้อมูลเพื่อนำกลับมาใช้ประโยชน์ในการประเมิลผลภายหลัง
ที่ผ่านมาซอฟต์แวร์ที่นิยมใช้ในการจัดเก็บข้อมูลมีหลายชนิด เช่น ไมโครซอฟต์เวิรด (Microsoft Word)
, ไมโครซอฟต์เอ็กเซล (Microsoft Excel)
การจัดเก็บข้อมูลก็ใช้วิธีการจัดเก็บในหน่วยความจำสำรองหรือฮาร์ดดิสค์ (Hard disk) ในเครื่องคอมพิวเตอร์บ้าง
จัดเก็บข้อมูลในผู้ให้บริการคลาวด์บ้าง จัดเก็บข้อมูลบเครื่องเซอร์เวอร์บ้าง มีความหลากหลายมากพอสมควร

กรณีใช้ซอฟต์แวร์ไมโครซอฟต์เอ็กเซลบันทึกลงในฮาร์ดดิสค์ของคอมพิวเตอร์
ข้อมูลถูกบันทึกในลักษณะตารางสองมิติ (2D table) ซึ่งมีความเก็บข้อมูลได้อย่างเป็นระบบ
สามารถนำมาใช้ในการประมวลเพื่อการประเมินผลได้ระดับหนึ่ง
แต่ด้วยคุณลักษณะเฉพาะของซอฟต์แวร์ไมโครซอฟต์เอ็กเซล เมื่อกำหนดความหมายของตารางสองมิติไปแล้ว
ก็จะไม่ยืดหยุ่น เมื่อนำไปใช้ในการประเมินผลก็ไม่สามารถนำไปใช้ได้หลากหลาย
การเก็บข้อมูลลงฮาร์ดดิสค์ มีความสะดวกในการทำงานส่วนตัวคนเดียว
แต่เมื่อจำเป็นต้องแลกเปลี่ยนและใช้ข้อมูลร่วมกัน กลายเป็นข้อจำกัดไม่สามารถทำได้อย่างมีประสิทธิภาพ
นอกจากนี้ การบันทึกข้อมูลผิดพลาด (human error) ก็เป็นสาเหตุสำคัญที่ทำให้
การติดตามผลและประเมินผลไม่สามารถทำได้อย่างมีประสิทธิภาพ

ในการนี้ผู้รับการประเมินได้ศึกษา เครื่องมือและกลไกที่เหมาะสมในการจัดเก็บข้อมูลติดตามผลและ
ประเมินผล เรียกว่า กิต (GIT) ซึ่งเป็นซอฟต์แวร์ที่ใช้ในระบบควบคุมรุ่นหรือเวอร์ชั่น (version) ของซอฟต์แวร์
โดยเริ่มแรกถูกพัฒนามาใช้สำหรับลินุกส์ (linux) โดยลินุส โตร์วัลดส์ ซึ่งเป็นซอฟต์แวร์ที่พัฒนาขึ้นใน
ลักษณะโอเพ่นซอส (Open Source) เกิดจากความร่วมมือกันพัฒนาให้ดีขึ้นเรื่อยๆ ของนักพัฒนาโปรแกรม
ไดเรกทอรี (director) หรือ โฟลเดอร์ (folder) ของกิต ทุกตัวจะมีการเก็บประวัติทั้งหมด
ข้อมูลที่ถูกจัดเก็บอยู่ในฮาร์ดดิสค์ จะถูกบันทึกความเปลี่ยนแปลงไว้อย่างเป็นระบบและมีประสิทธิภาพ
เมื่อเกิดความผิดพลาด เช่น บันทึกข้อมูลผิด ต้องการจับกลับไปใช้ข้อมูลในอดีต เช่น ของชั่วโมงที่แล้ว ของเดือนที่แล้ว
ของปีที่แล้ว ก็สามารถทำได้ง่าย เนื่องจาก กิตมีการบันทึกประวัติเอาไว้ทั้งหมด

นอกจากนี้ การใช้กิตใช้การแลกเปลี่ยนและใช้ข้อมูลร่วมกันในองค์กรก็สามารถทำได้ง่าย
ถึงแม้ขณะทำงานหรือขณะที่กำลังบันทึกแก้ไขข้อมูลจะทำแบบออฟไลน์ (ไม่จำเป็นต้องต่ออินเทอร์เน็ท)
เมื่อบันทึกข้อมูลเสร็จก็สามารถอัพเดทความเปลี่ยนแปลงทั้งหมดที่เกิดขึ้นบนเครื่องคอมพิวเตอร์ของตน
ไปยังเครื่องคอมพิวเตอร์เซอร์เวอร์ หรือเครื่องคอมพิวเตอร์บนคลาวด์ ได้อย่างรวดเร็วและมีประสิทธิภาพ
ต่างจากการบันทึกข้อมูลบนคลาวด์โดยทั่วไปซึ่งจำเป็นต้องต่ออินเทอร์เน็ทอยู่ตลอดเวลา
ซึ่ทำให้ผู้อื่นที่ต้องการแลกเปลี่ยนและใช้ข้อมูลร่วมกันสามารถนำข้อมูลอัพเดทล่าสุดนั้นไปใช้
ในเครื่องคอมพิวเตอร์ของตนได้อย่างสะดวกรวดเร็ว โดยไม่จำเป็นต้องมีซอฟต์แวร์เครื่องมืออื่นที่มีราคาแพง

กิต (GIT) เป็นซอฟต์แวร์ที่มีลักษณะเป็นมาตรฐานเปิด เป็นมาตรฐานสากล ไม่ขึ้นอยู่กับผลิตซอฟต์แวร์
รายใดรายหนึ่ง ทำให้เกิดมีผู้ผลิตผู้พัฒนาซอฟต์แวร์กิตหลายผู้ผลิต แต่สามารถทำงานร่วมกันได้เนื่องจาก
เป็นมาตรฐานเปิด เป็นมาตรฐานสากล เราสามารถเปลี่ยนซอฟต์แวร์เป็นผู้ผลิตอื่นได้ภายหลังตามความต้องการ
โดยไม่กระทบต่อการทำงานในการเก็บบันทึกและประมวลข้อมูล เช่นเดียวกับ อีเมล์ (email) 
ซึ่งก็เป็นมาตรฐานสากล เป็นมาตรฐานสากล เราสามารถส่งอีเมล์จากซอฟต์แวร์ของใครก็ได้ 
ผู้รับการสามารถใช้อีเมล์ของใครก็ได้ เช่น สามารถส่งอีเมล์จาก gmail ไปยัง yahoo mail หรือ outlook
ได้ ในขณะที่ซอฟต์แวร์ช่องทางการสื่อสารอื่นทำไม่ได้ เช่น ถ้าใช้ LINE ก็รับส่งข้อมูลกันได้เฉพาะ
ภายใน แอ๊พ LINE เท่านั้น ถ้าใช้ FACEBOOK ก็สามารถรับส่งข้อมูลกันภายใน แอ๊พเดียวกัน LINE ไม่สามารถส่ง
ข้อมูลโดยตรงให้ FACEBOOK ได้ กิต ก็เช่นเดียวกันเป็นซอฟต์แวร์ที่มีลักษณะเป็นมาตรฐานเปิด
ไม่ขึ้นอยู่กับผู้ผลิตรายใดรายหนึ่ง

เมื่อใช้ กิต(GIT) ประกอบกับมาตรฐานอาร์ดีเอ็ฟ (RDF) ก็จะทำให้มี
มาตรฐานรูปแบบวิธีการและเครื่องมือกลไก ที่เหมาะสม
สำหรับติดตามผลและประเมินผลโครงการได้อย่างมีประสิทธิภาพ กล่าวคือ การบันทึกข้อมูลสามารถทำได้
บนเครื่องคอมพิวเตอร์ปกติที่ใช้อยู่ และสามารถแลกเปลี่ยนและใช้ข้อมูลร่วมกันได้ง่ายระหว่างผู้เกี่ยวข้อง
โดยการบันทึกข้อมูลเกี่ยวกับการติดตามผลโครงการในรูปแบบ RDF
และบันทึกและใช้ข้อมูลร่วมกันทั้งองค์กรด้วย ซอฟต์แวร์ กิต
เมื่อต้องการวิเคราะห์ คัดสรรและดึงข้อมูลก็ใช้ความสามารถของ RDF โดยใช้ภาษา SPARQL
ซึ่งเป็นภาษามาตรฐานเพื่อใช้ในการคัดสรรค์ข้อมูลที่ต้องการได้อย่างมีประสิทธิภาพ

ตัวอย่างการติดตามผลและประเมินผลโครงการ โดยใช้ GIT และ RDF
